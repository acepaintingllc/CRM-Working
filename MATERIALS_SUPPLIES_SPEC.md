# Materials / Supplies Module Spec

## Purpose

This document defines how paint, primer, and supplies should work in the estimator system.

The goal is to keep materials and supplies visible, flexible, and accurate while supporting grouped rollups, shared costs, and room-level allocation.

## Module Goal

The materials / supplies module should:

- calculate raw paint and primer needs from scope items
- group shared colors and products together before rounding
- allocate rounded material cost back to rooms and scope items
- calculate supplies cost separately from paint and primer
- support both shared and direct supply cost
- keep all costs visible for debugging and pricing control

## Core Philosophy

Paint, primer, and supplies are separate.
The system should not treat all material cost as one blended number.

Keep a clear distinction between:

- paint gallons
- primer gallons
- grouped purchase quantities
- direct supply cost
- shared supply cost

## Material / Supply Buckets

### 1. Paint

Used for finish coats.

Should support:

- raw gallons by scope item
- grouped gallon rollup by shared color/product
- rounded purchase quantity
- cost allocation back to source rows

### 2. Primer

Used for priming.

Should support:

- raw gallons by scope item
- grouped gallon rollup if shared
- rounded purchase quantity
- cost allocation back to source rows

Primer remains separate from paint in both logic and reporting.

### 3. Per-color supplies

Shared by all scope items using the same color or product group.

Examples:

- brush cost
- roller cover cost
- tray/liner cost
- other color-specific consumables

Behavior:

- calculated once per color group
- split across rooms or scope items using that color

### 4. Area-based supplies

Driven by square footage.

Examples:

- small drywall touchups
- nail holes
- tape
- caulk
- misc sundries

Behavior:

- calculated from effective area
- applied to the relevant scope item
- rolled into room totals

### 5. Manual supplies

Used for one-off consumables.

Examples:

- special masking
- plastic
- extra sanding materials
- unusual prep materials

Behavior:

- manually entered when needed
- directly assigned to room, scope item, or estimate version depending on the case

## Main Entities

### Material Requirements

Purpose:

- store raw paint and primer needs generated by scope items

Notes:

- one row per source requirement
- linked to estimate version
- linked back to source scope item
- stores raw gallons before grouping and rounding

Examples:

- bedroom walls paint raw gallons
- garage ceiling primer raw gallons

### Material Purchase Groups

Purpose:

- group shared material requirements before rounding and costing

Notes:

- grouped by estimate version plus material/product/color combination
- sum raw gallons
- round once
- determine grouped material cost
- allocate cost back down

Examples:

- all wall paint using one color
- all primer using one primer product

### Supply Requirements

Purpose:

- store supply cost requirements from scope items or color groups

Notes:

- support direct and shared supply logic
- support per-color supplies
- support area-based supplies
- support manual supplies

## Material Flow

### Step 1: scope items calculate raw gallons

Each relevant scope item calculates:

- raw paint gallons
- raw primer gallons

Based on:

- effective area
- product coverage assumptions
- coat and prime logic
- overrides if present

### Step 2: create material requirements

Each scope item creates material requirement rows including:

- estimate version
- source type
- source id
- material type
- product/color reference
- raw gallons

### Step 3: group shared materials

Requirements using the same product/color are grouped together.

Purpose:

- avoid rounding the same color separately across multiple rooms
- support more realistic purchasing behavior

### Step 4: round grouped quantities

Grouped gallons are rounded once at the grouped purchase level.

Examples:

- `2.2` gallons -> `3` gallons
- `4.1` gallons -> `5` gallons

Exact purchase-unit logic can be refined later.

### Step 5: determine grouped material cost

Each grouped material purchase calculates:

- rounded purchase quantity
- unit cost
- total grouped material cost

### Step 6: allocate cost back to source rows

Grouped material cost is allocated back to source scope items.

Likely default:

- proportional by raw gallons contribution

## Supplies Flow

### Step 1: determine supply type

Supply cost can come from:

- per-color shared supplies
- area-based supplies
- manual supplies

### Step 2: calculate per-color supplies

Generated once per color/product group.

Examples:

- one brush cost
- one roller cover cost
- one tray/liner cost

If multiple rooms use the same color group, those supply costs are split across those contributors.

### Step 3: calculate area-based supplies

Calculated directly from effective area.

Examples:

- tape cost per square foot
- caulk cost per square foot
- nail-hole or minor patch supply allowance per square foot

These are assigned directly to the source scope item or room.

### Step 4: add manual supplies

Manual supply rows should support:

- description
- cost
- room, scope item, or estimate version assignment

### Step 5: roll supply cost into totals

Supply cost should roll into:

- scope totals
- room totals
- estimate totals

Supply cost should remain visible separately from labor and paint/primer gallons.

## Suggested Internal Outputs

### Material outputs

- raw paint gallons
- effective paint gallons
- grouped paint gallons
- rounded paint purchase quantity
- allocated paint cost
- raw primer gallons
- effective primer gallons
- grouped primer gallons
- rounded primer purchase quantity
- allocated primer cost

### Supply outputs

- shared per-color supply cost
- allocated per-color supply cost
- area-based supply cost
- manual supply cost
- total supply cost

## Rollup Behavior

### Scope item level

Each scope item should be able to show:

- raw gallons
- effective gallons
- allocated material cost
- direct supply cost
- allocated shared supply cost
- total material/supply cost

### Room level

Each room should roll up:

- allocated paint cost
- allocated primer cost
- direct supplies
- shared supply allocations
- total material/supply cost

### Estimate version level

Each estimate version should roll up:

- total paint gallons
- total primer gallons
- total grouped material purchases
- total supplies cost
- final material/supply cost

## Overrides

Possible override types:

- raw gallons override
- effective gallons override
- grouped purchase override
- supply cost override
- manual supply add/remove

All overrides should remain visible internally.

## Catalog / Settings Dependencies

This module will likely depend on:

- Products
- product coverage values
- supply catalog
- per-color supply defaults
- area-based supply rates
- pricing package defaults later

The structure should support these without hardcoding them into individual scope modules.

## Future Features

Leave room for:

- good / better / best product packages
- package-driven supply assumptions
- different purchase unit types
- more advanced allocation methods
- product substitution history
- historical snapshot protection

## Open Decisions

- exact purchase rounding behavior by product type
- default allocation method for per-color supplies
- whether some supplies should be estimate-level instead of room/scope-level
- how deeply product/package defaults should drive supply assumptions
- whether manual supplies should belong to room, scope item, or estimate version depending on case

## Design Rules

- keep paint, primer, and supplies separate
- round grouped materials only once
- allocate grouped costs back visibly
- keep supply logic visible and flexible
- avoid hidden blended material costs
- support both shared and direct cost patterns
